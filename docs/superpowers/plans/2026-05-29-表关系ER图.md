# 表关系 ER 图 Implementation Plan

> **For agentic workers:** 用 superpowers:executing-plans 逐任务实现。Steps 用 checkbox 跟踪。
> Spec 单一来源：`/Users/java/obsidian/01 Engineering/axon-link-server/表关系ER图-设计.md`

**Goal:** 在 SQL 巡检大菜单下新增「ER 图」子页：扫 uat 目标库全部表，按「键包含」启发式推断隐式外键关系（3 级置信度），落库可人工修正，前端聚焦模式手绘 SVG 画布呈现，支持导出关系清单 Excel。

**Architecture:** 后端全库批量 2 条 SQL 拿索引+列 → 内存推断 → upsert `dii_er_relation`（保留人工 CONFIRMED/IGNORED）；REST 暴露 rebuild/tables/graph/status/export；前端 `DaoErDiagram.vue` 聚焦某表画 N 跳子图，原生 SVG + `<g transform>` 平移缩放、节点拖动、置信度过滤。

**Tech Stack:** Java 17 / Spring JdbcTemplate / Apache POI（已有）/ Vue 3 `<script setup>` / 原生 SVG（零新依赖）。

    id              BIGINT       AUTO_INCREMENT PRIMARY KEY,

    INDEX idx_env_from (env, from_table),
    INDEX idx_env_to   (env, to_table),
    INDEX idx_env_conf (env, confidence, status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='表关系 ER 推断结果（隐式外键）';

- [ ] Step 1: 写失败单测 `ErInferenceServiceTest`：
  - `compositeKeyAllMatch_returnsHigh`：A 联合键 (a_id,b_id)，B 含两列且非 B 键 → HIGH
  - `singleDistinctColumn_returnsMedium`：A 单列键 acct_no，acct_no 仅出现 3 张表(<=5) → MEDIUM
  - `singleCommonColumn_returnsLow`：A 单列键 id，id 出现 50 张表(>5) → LOW

  - `partialColumnMatch_skipped`：B 只含 S 的部分列 → 不产出
- [ ] Step 2: `mvn test -Dtest=ErInferenceServiceTest` 验证 FAIL
- [ ] Step 3: 实现 `infer(...)`：双层循环 + confidence 判定（见 spec 算法节）
- [ ] Step 4: `mvn test -Dtest=ErInferenceServiceTest` 验证 PASS

核心实现骨架：
```java
public List<ErRelation> infer(Map<String,List<ErKeySet>> keySets,
                              Map<String,Set<String>> tableColumns,
                              Map<String,Integer> columnTableCount,
                              int distinctMax, Set<String> blacklist) {
    List<ErRelation> out = new ArrayList<>();
    for (var eA : keySets.entrySet()) {
        String tableA = eA.getKey();
        for (ErKeySet s : eA.getValue()) {
            Set<String> scol = new HashSet<>(s.getColumns());
            for (var eB : tableColumns.entrySet()) {
                String tableB = eB.getKey();
                if (tableB.equals(tableA)) continue;
                if (!eB.getValue().containsAll(scol)) continue;
                if (isSameAsAnyKey(scol, keySets.get(tableB))) continue; // 排除同实体
                String conf;
                if (s.getColumns().size() >= 2) conf = "HIGH";
                else {
                    String col = s.getColumns().get(0);
                    boolean distinct = columnTableCount.getOrDefault(col, 0) <= distinctMax;
                    boolean black = blacklist.contains(col);
                    conf = (distinct && !black) ? "MEDIUM" : "LOW";
                }
                out.add(new ErRelation(tableA, tableB, s.getColumns(),
                        s.getKeyType(), s.getColumns().size(), conf));
            }
        }
    }
    return out;
}
```
