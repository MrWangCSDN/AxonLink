  否则给 tool_calls。避免了标题生成调用被错误地当成"该发起工具调用"处理。

## B2：后端本机启动 —— 失败，根因与本分支无关（预存环境限制）

尝试命令：

```bash
cd <backend worktree>
AI_OPENCODE_ENABLED=true AI_OPENCODE_PROVIDER_ID=poc AI_OPENCODE_MODEL_ID=mock-model mvn spring-boot:run
```

**结果：`BUILD FAILURE`，启动阶段即失败**，根因栈（节选）：

```
Caused by: org.springframework.beans.BeanInstantiationException: Failed to instantiate
  [com.zaxxer.hikari.HikariDataSource]: Factory method 'diiResultDataSource' threw exception
  with message: Failed to initialize pool: Communications link failure
Caused by: com.zaxxer.hikari.pool.HikariPool$PoolInitializationException: Failed to initialize
  pool: Communications link failure
Caused by: com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
  at com.axonlink.ai.daoindex.config.DaoIndexDataSourceConfig.diiResultDataSource(...)
Caused by: java.net.SocketTimeoutException: Read timed out
```

**根因**：`DaoIndexDataSourceConfig.diiResultDataSource`（DII 结果库）是一个无条件装配的
`@Configuration` Bean（未加 `@ConditionalOnProperty`），构造 `HikariDataSource` 时用的是
Hikari 默认 fail-fast 行为（未设置 `initializationFailTimeout`），目标地址
`jdbc:mysql://21.64.203.16:3306/benchmarkdb`（`DII_RESULT_URL` 默认值）是内网地址，
本机（外网开发机）连不通，pool 初始化直接抛异常，导致整个 Spring 容器起不来。

**判定：与本分支（feat/opencode-deep-analysis）无关**——已核实：
- `git diff $(git merge-base main HEAD) HEAD --stat` 显示本分支只改了
  `ai/opencode/*`（新增包）+ `config/WebMvcConfig.java`（9 行，只是把全局异步超时上限
  加了个 `Math.max` 分支，不影响其它端点）+ `application.yml`（新增 `ai.opencode.*` 段）
  + 文档/artifacts。`DaoIndexDataSourceConfig.java` 完全不在本分支的改动范围内。
- 这是该项目在"外网本地开发机、无 VPN/内网直连"场景下的**预存限制**，与是否开启
  `AI_OPENCODE_ENABLED` 无关——即使 `AI_OPENCODE_ENABLED=false` 起服务，同样会在这一步失败。
- `application-local.yml.example` 里"环境 A：外网本地"配置块只覆盖了 Neo4j / workspace-roots /
  flowtran datasource，没有覆盖 DII 结果库，说明这确实是该项目目前没有为纯外网本机准备的一个盲区，
  不是本次改动引入的回归。

按任务指引"只试 5 分钟，别深陷"：已确认没有任何环境变量能让这个 Bean 绕过（无
`enabled` 开关、无 `initializationFailTimeout` 覆盖入口），需要代码改动（加
`@ConditionalOnProperty` 或设置 lenient pool init）才能解决，这超出本次"端到端联调"
任务范围（不应该为了联调去改无关模块的生产代码），故未继续尝试，直接按指引转 B4，
B3 挂起并列入下方"留待内网环境"清单。

如内网默认开启了 Security，需要 `--axon-link.security.enabled=false` 或对应 env 关掉，
否则要先登录拿 session cookie。

`server.port` 实际配置的是 **8123**（除非 env `SERVER_PORT` 覆盖），内网执行时请用真实端口。

清单：

   —— 验证 NDJSON 行完整：`start` → `tool`/`delta` 交替 → `done`。

   的实际行为（抛错被 catch 落 `error` 事件，还是返回空上下文继续往下走）——两种行为都算
   有效验证，把实际现象记录进本文件即可，不需要预判。
3. 杀掉 opencode serve 后再发起同样请求 → 预期收到 `fallback` 事件后走路径①单轮结果；
   若路径①依赖的 Neo4j/结果库在内网也不可达，预期在 `fallback` 之后紧跟 `error`

   `"降级分析也失败了：..."` 的 error 事件）——同样是"记录实际现象"而非预判对错。
4. `AI_OPENCODE_ENABLED=false` 重启后端 → 预期与场景 3 一致（未启用直接走 fallback 分支，
